Comprehensive

Samples contain full open source code and details on logic, as well as "keep it simple stupid" as an ingrained principle

Simple

Code doesn't have to be complicated. Examples should be comprehensive but not turing-complete. A sample should show you how to paint a red fence, but it doesn't need to show you how to paint it green with stripes.

Robust

These experiments work across as many browsers as possible (dependant APIs permitting). With a focus on ignoring Internet Explorer, Safari, and Firefox, we can cover all use-cases with minimal browser vendor quirks.
